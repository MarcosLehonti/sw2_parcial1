Paso 1: Ingreso de la URL

El primer paso para iniciar un análisis de vulnerabilidades es proporcionar la dirección web (URL) del sitio o aplicación que deseas escanear. Es importante que la URL esté escrita correctamente, incluyendo el protocolo http:// o https://.

Por ejemplo, una URL válida sería: https://www.ejemplo.com. En cambio, si escribes solamente www.ejemplo.com sin el protocolo, el sistema intentará añadir http:// por defecto, pero lo ideal es que siempre incluyas el protocolo para evitar confusiones.

Una vez que hayas escrito la URL en el campo correspondiente, haz clic en el botón “Iniciar escaneo”. El software validará la dirección y, si es correcta, comenzará a ejecutar las pruebas de seguridad sobre la página indicada.

Dependiendo del tamaño y la complejidad del sitio web, el análisis puede tardar algunos segundos o incluso varios minutos. Durante este tiempo, podrás observar un indicador de progreso en la pantalla que te informará que el proceso se está ejecutando correctamente.

Paso 2: Consola de escaneo

Después de presionar el botón “Iniciar escaneo”, se abrirá la consola de escaneo. Esta sección es fundamental porque te muestra en tiempo real el progreso del análisis que el sistema está realizando sobre la URL que ingresaste previamente.

En la consola se irán desplegando mensajes y registros que indican qué pruebas de seguridad se están ejecutando en ese momento. Esto incluye, por ejemplo, la verificación de cabeceras de seguridad, posibles vulnerabilidades comunes (como inyecciones SQL o XSS), y el estado de respuesta del servidor.

El objetivo de esta vista es que puedas monitorear en vivo cómo avanza el escaneo. De esta manera, si ocurre algún error (como una URL inválida o un tiempo de espera agotado), lo sabrás al instante.

Dependiendo del tamaño del sitio y de la complejidad de las pruebas, el contenido de la consola puede actualizarse durante varios segundos o minutos. Al finalizar, el sistema te notificará que el escaneo ha concluido y automáticamente te llevará a la siguiente sección: el resumen de vulnerabilidades detectadas.

Paso 3: Resumen de vulnerabilidades

Una vez que el escaneo haya concluido, se generará automáticamente un resumen detallado con los hallazgos encontrados en la URL analizada. Este resumen se presenta en forma de tabla para que la información sea clara, estructurada y fácil de interpretar.

Cada fila de la tabla representa una vulnerabilidad detectada. Se incluirán datos como el tipo de vulnerabilidad (por ejemplo: inyección SQL, XSS, cabeceras inseguras, etc.), una breve descripción del problema y su nivel de severidad.

Los niveles de severidad se categorizan en:

  • Alta: Vulnerabilidades críticas que representan un riesgo inmediato y deben solucionarse con urgencia.
  • Media: Vulnerabilidades importantes pero no críticas; requieren atención para evitar que se conviertan en un riesgo mayor.
  • Baja: Riesgos menores o configuraciones no óptimas que conviene corregir para fortalecer la seguridad.

Este resumen es esencial porque te ofrece una visión global de la seguridad de tu sitio web o aplicación, permitiéndote priorizar las acciones de corrección de acuerdo con la gravedad de los hallazgos.

Paso 4: Correcciones sugeridas

En esta sección encontrarás un conjunto de medidas recomendadas para mitigar y solucionar cada una de las vulnerabilidades que fueron detectadas durante el escaneo. Estas sugerencias están diseñadas para guiarte en la implementación de mejores prácticas de seguridad.

Por cada vulnerabilidad listada en el resumen anterior, se mostrarán pasos o recomendaciones específicas. Por ejemplo:

  • Actualizar la versión del servidor o framework a una más segura.
  • Corregir configuraciones inseguras en cabeceras HTTP.
  • Implementar validación de entradas para prevenir inyecciones SQL o XSS.
  • Configurar contraseñas y llaves de acceso con mayor fortaleza.
  • Aplicar parches o actualizaciones de seguridad recomendadas por proveedores.

El propósito de esta sección es ayudarte a pasar de la detección a la acción, brindándote la información necesaria para reforzar la seguridad de tu aplicación o sitio web. Aunque algunas correcciones pueden requerir conocimientos técnicos avanzados, seguir estas recomendaciones reducirá significativamente los riesgos de ataque.

Una vez aplicadas las correcciones, se recomienda volver a ejecutar el escaneo para verificar que las vulnerabilidades hayan sido resueltas correctamente.

Paso 5: Descargar reportes

Finalmente, puedes descargar los resultados en dos formatos:

  • Reporte PDF de vulnerabilidades: contiene el listado completo de hallazgos.
  • Reporte PDF de correcciones: incluye las recomendaciones para solucionar los problemas detectados.

De esta manera tendrás documentación clara tanto de las vulnerabilidades encontradas como de las acciones necesarias para corregirlas.
